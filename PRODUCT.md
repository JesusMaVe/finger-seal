# Product

## Register

product

## Users

Individual developers working on personal projects, side projects, or weekend coding sessions. They're comfortable with SQL but don't want to fight their tools—they want fast, unobtrusive access to multiple database types (PostgreSQL, MySQL, SQLite, Oracle) from a single interface. They typically have a laptop open on a coffee table or home desk, context-switching between a code editor and the database UI.

## Product Purpose

Finger Seal is a desktop database management tool that lets developers explore schemas, write and run queries, manage connections, and monitor database health—all from a clean, low-friction UI. Success means the developer reaches for Finger Seal instead of raw CLIs or clunky legacy tools. It exists because existing tools are either heavy (DBeaver, DataGrip), ugly (pgAdmin), or web-only. Finger Seal is native, lightweight, and stays out of the way.

## Brand Personality

Modern, quiet, capable. Three words: **sharp, calm, confident**.

Like a well-designed developer tool that doesn't need to shout. The interface earns trust through clarity and responsiveness, not through decorative flair. Tone is neutral and technical—labels say what they mean, copy wastes no words.

References: Supabase (clean, generous whitespace, teal accent, approachable), Linear (tight typography, confident interaction design, no visual noise).

## Anti-references

- **pgAdmin / DBeaver**: classic legacy database tools—cluttered, inconsistent spacing, tiny fonts, 90s icons, too many toolbars. Anything that triggers "corporate IT tool" energy.
- **Grafana / Datadog**: dark, dense, overwhelming dashboards. Too much information competing for attention.
- **Forced dark mode**: dark themes are fine, but the user decides. Not locked in.
- **SaaS-cream, AI-chat bubbles**: finger seal is not a marketing page or a chatbot.

## Design Principles

1. **Function first.** Every pixel earns its place. If a visual element doesn't help the user query, explore, or understand their database, it's distraction. But "function" includes legibility, hierarchy, and confidence at a glance—not just raw pixels.
2. **Stay out of the way.** The user's data is the hero. Chrome (sidebars, headers, badges) recedes. The interface is a frame, not a decoration.
3. **System rhythm, not imposed theme.** Respect the OS. Light and dark modes are system-adaptive, not designer-chosen. The manual toggle is a convenience, not a statement.
4. **Teal confidence, not blue safety.** Accent is a muted teal—warm enough to feel crafted, cool enough to stay professional. It's an accent, not a brand splash.
5. **Editorial quiet.** Generous whitespace, consistent rhythm, no hero metrics or gradient text. Typography does the heavy lifting for hierarchy.

## Accessibility & Inclusion

No formal requirements until someone reports a need. The existing system-adaptive theme and generous font sizes provide a reasonable baseline.
